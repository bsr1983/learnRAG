# 学习计划执行指南

## 📋 计划概述

本学习计划将 RAG + 智能体编排系统的学习拆分为 **21 天**的详细可执行步骤，每天都有明确的学习目标、实践任务和检查点。

## 🎯 核心目标

通过 3 周的系统学习，深入理解 RAG 技术栈，掌握从嵌入模型到评测系统的完整流程，并具备自研编排框架的能力。

## 📅 执行步骤总览

### 第 1 周：RAG 系统核心组件入门

| 天数 | 主题 | 核心任务 | 代码示例 |
|------|------|----------|----------|
| Day 1-2 | 语义嵌入与向量基础 | 学习 embedding 模型，计算相似度 | `embeddings/basic_embedding_demo.py` |
| Day 3-4 | 向量数据库 | 使用 Qdrant 存储和检索向量 | `storage/qdrant_demo.py` |
| Day 5-7 | 构建最小 RAG Demo | 实现完整的 RAG 流程 | `retrieval/basic_rag_demo.py` |

### 第 2 周：增强与结构化生成

| 天数 | 主题 | 核心任务 | 代码示例 |
|------|------|----------|----------|
| Day 8-10 | RAG-Fusion | 实现多查询融合检索 | `retrieval/rag_fusion_demo.py` |
| Day 11-13 | 结构化输出 | 使用 DSPy/Guidance | `llm/structured_output_demo.py` |
| Day 14 | 系统整合 | 整合所有组件 | `app/integrated_rag_system.py` |

### 第 3 周：评测与编排体系建设

| 天数 | 主题 | 核心任务 | 代码示例 |
|------|------|----------|----------|
| Day 15-17 | Ragas 评测 | 自动化评测系统 | `evaluation/ragas_eval_demo.py` |
| Day 18-19 | ChatArena 评测 | 对话质量评测 | - |
| Day 20-21 | 自研编排器 | 设计并实现编排框架 | `orchestration/minimal_orchestrator.py` |

## 🚀 如何执行

### 步骤 1: 环境准备（第 1 天开始前）

```bash
# 1. 创建虚拟环境
python -m venv venv
source venv/bin/activate

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置环境变量
# 编辑 .env 文件，配置 OPENAI_API_KEY 等

# 4. 启动 Qdrant
docker run -p 6333:6333 qdrant/qdrant
```

### 步骤 2: 按天执行学习计划

#### 每日执行流程

1. **阅读学习目标**（README.md 中对应天的内容）
2. **理论学习**（阅读推荐的论文/文档）
3. **运行示例代码**（理解代码逻辑）
4. **修改实验**（尝试不同参数，观察效果）
5. **记录笔记**（在 `docs/learning_notes.md` 中记录）
6. **完成检查点**（确保达到学习目标）

#### 示例：Day 1-2 执行流程

```bash
# 1. 阅读 Day 1-2 的学习内容（README.md）
# 2. 阅读 Sentence-BERT 论文摘要
# 3. 运行示例代码
python embeddings/basic_embedding_demo.py

# 4. 修改代码，测试不同文本
# 5. 记录学习笔记
# 6. 完成检查点
```

### 步骤 3: 每周复盘

每周结束时：
1. 回顾本周学习内容
2. 完成本周的检查清单（`docs/learning_checklist.md`）
3. 总结遇到的问题和解决方案
4. 规划下周的学习计划

### 步骤 4: 最终整合

第 3 周结束时：
1. 运行完整的集成系统（`app/integrated_rag_system.py`）
2. 完成所有检查清单
3. 编写学习总结
4. 规划后续学习方向
